# arm26_paper_loaded_brd_elbow_research.osim - How It Works

This document explains the recommended Stage-2 OpenSim model:

```text
Model/arm26_paper_loaded_brd_elbow_research.osim
```

Scope boundary: this document is movement/biomechanics-focused. Vision topics
are only referenced as downstream Stage-3 usage of the labels generated from
this model.

This is the best current model for the thesis scope: repetitive loaded elbow
flexion with a fixed/stabilized shoulder.

The model is not a generic full upper-limb model. It is a focused biomedical
model designed to answer one main question:

```text
How do validated elbow flexors share load during a repetitive 2 kg elbow-curl
motion generated by Stage 1?
```

---

## 1. Model Purpose

The model exists to turn Stage-1 joint-angle motion into Stage-2 biomechanical
outputs:

- elbow inverse-dynamics torque;
- muscle moment arms;
- muscle forces and activations from Static Optimization;
- loaded-curl demand under a real 2 kg dumbbell body;
- data features for later fatigue/AI analysis.

It is based on the OpenSim arm26 model, then improved for the thesis task by
adding a realistic hand load and a validated brachioradialis path.

---

## 2. Model Lineage

The model was created in stages:

```text
stock arm26
    -> arm26_paper_loaded.osim
    -> arm26_paper_loaded_brd.osim
    -> arm26_paper_loaded_brd_elbow_research.osim
```

Meaning:

| Model | Role |
|-------|------|
| `arm26_paper_loaded.osim` | loaded model, but BRD not yet validated |
| `arm26_paper_loaded_brd.osim` | full validated BRD + dumbbell model |
| `arm26_paper_loaded_brd_elbow_research.osim` | recommended elbow-study model; same elbow/dumbbell mechanics, shoulder muscles removed |

The full BRD model is preserved for traceability. The elbow-research model is
the safer production choice for this fixed-shoulder elbow-flexion study.

## 3. Main Mechanical Structure

The model has:

```text
coordinates = 2
muscles     = 11
bodies      = 4
gravity     = 0 -9.8066 0
```

The two coordinates are:

| Coordinate | Meaning | Stage-1 usage |
|------------|---------|---------------|
| `r_shoulder_elev` | shoulder elevation | fixed at about 20 deg |
| `r_elbow_flex` | elbow flexion | driven by the generated minimum-jerk curl |

The Stage-1 `.mot` files therefore only need:

```text
time    r_shoulder_elev    r_elbow_flex
```

---

## 4. Muscle System

The model contains the elbow muscles and the validated BRD path. The four
shoulder muscles (DELT_ant/post, PECT, LAT) have been **removed** — this is an
elbow-focused model, and the fixed shoulder is held by the `shoulder_assist`
coordinate actuator (stock arm26 has no shoulder muscles either).

| Group | Muscles | Role |
|-------|---------|------|
| elbow flexors | `BIClong`, `BICshort`, `BRA`, `BRD_hand` | main curl producers |
| elbow extensors | `TRIlong`, `TRIlat`, `TRImed` | antagonists / co-contraction |
| actuators | `shoulder_assist`, `elbow_assist` | residual support and stabilization |

(7 muscles total: 4 flexors + 3 extensors.)

The key improvement is `BRD_hand`. It is tuned to behave like human
brachioradialis rather than like a duplicate brachialis.

---

## 5. Validated Elbow Flexor Hierarchy

OpenSim API validation gives the following peak elbow-flexion moment arms:

```text
BRD_hand           7.51 cm @ 115 deg
BIClong/BICshort   4.94 cm @ 100 deg
BRA                2.40 cm @ 110 deg
```

So the validated flexor order is:

```text
BRD_hand > BIClong/BICshort > BRA
```

This is biomechanically important. In human upper-limb data, brachioradialis has
a large elbow-flexion moment arm, especially at higher elbow flexion angles. If
BRD is missing or too weak geometrically, Static Optimization can overload the
biceps and brachialis. This model fixes that problem.

Validation result:

```text
Murray target: 7.0-9.0 cm peak near 100-118 deg
model result:  7.51 cm peak at 115 deg
verdict:       validated
```

---

## 6. Why the Shoulder Muscles Were Removed

The four shoulder muscles (`DELT_ant`, `DELT_post`, `PECT`, `LAT`) have been
**removed** from this elbow-focused model. Reasons:

- **None of them cross the elbow**, so they contribute nothing to elbow-flexion
  loading — the variable of interest.
- Their simplified paths were non-physiological (e.g. LAT operated at ~0.29×
  optimal fiber length), which risked contaminating Static Optimization.
- Stock arm26 has no shoulder muscles either; removing them returns to that clean
  baseline plus the validated elbow extensions.

The bad option would have been to hide the problem by changing one physiology
number (`optimal_fiber_length`, `tendon_slack_length`) just to make the output
look better — not a real biomedical fix.

```text
For this elbow-only study: remove the shoulder muscles.
For a future shoulder study: rebuild and validate proper shoulder muscle paths.
```

The fixed shoulder is held by the `shoulder_assist` coordinate actuator, while the
elbow muscle mechanics remain unchanged and validated.

## 7. Dumbbell Load Model

The model includes a real 2 kg body:

```text
Body name:    load_2kg
mass:         2.000 kg
mass center:  0 0 0
inertia:      0.0087 0.0009 0.0087 0 0 0 kg*m^2
```

The dumbbell is welded to the hand/forearm segment:

```text
load_weld translation = 0.028 -0.285 0.082 m
load_weld rotation    = 1.570796 0 0 rad
```

This is better than manually injecting a fake elbow torque. Gravity acts on the
2 kg body, and the elbow load emerges from the model geometry.

Expected load torque over the Stage-1 curl:

```text
low elbow angle:      about -3.7 to -4.4 N*m
mid curl:             about -5.65 N*m peak
high elbow flexion:   about -3.5 N*m
```

This matches the physics estimate:

```text
2 kg * 9.81 m/s^2 * about 0.29 m = about 5.7 N*m
```

## 8. How It Connects To Stage 1

Stage 1 writes a motion file such as:

```text
Data/paper_minjerk_fatigue_10cycles.mot
```

That file drives only the two model coordinates:

```text
r_shoulder_elev = fixed stabilization angle
r_elbow_flex    = fatigue-aware minimum-jerk elbow curl
```

Then Stage 2 uses this model to compute:

- load torque from gravity and the dumbbell body;
- elbow moment arms from muscle geometry;
- muscle activations and forces from Static Optimization;
- residual actuator use.

## 10. What Is Already Fixed

| Issue | Current state |
|-------|---------------|
| BRD moment arm too small | fixed and validated against Murray |
| BRD physiology values | set to Holzbaur/Murray-consistent values with calibrated tendon slack |
| dumbbell mass | real 2 kg body, not synthetic torque |
| dumbbell inertia | realistic compact-dumbbell inertia tensor |
| dumbbell grip | moved to palm/MCP region |
| dumbbell orientation | handle aligned across the palm and steady through curl |
| shoulder-muscle contamination | DELT_ant/post, PECT, LAT removed for elbow-only research model |

## 11. Remaining Limitations

The model is strong for loaded elbow-flexion research, but it is still scoped.

- It is a 2-DOF arm26 model, not a full shoulder-scapula-trunk model.
- The shoulder muscles are removed, not rebuilt. That is correct for the elbow
  study but not sufficient for a shoulder-focused study.
- Original Arm26 `.vtp` geometry files are missing from the workspace, so OpenSim
  GUI visuals are incomplete until the `Geometry` folder is added.
- Inverse Dynamics and Static Optimization still need to be run on the final
  motion to confirm BRD load sharing and residual actuator levels.
- The model is validated against literature moment arms and load physics, not
  against subject-specific EMG or motion capture.

## 12. Production Use

Use this pair for the current pipeline:

```text
Model/arm26_paper_loaded_brd_elbow_research.osim
Data/paper_minjerk_fatigue_10cycles.mot
```

For a larger dataset, use the Stage-1 multi-subject `.mot` files with the same
model:

```text
Data/Subjects/subject_01.mot
Data/Subjects/subject_02.mot
...
```

After OpenSim ID/SO, check:

- elbow inverse-dynamics torque peaks near the expected loaded-curl range;
- `BRD_hand` force and activation are nonzero;
- biceps and brachialis are not unrealistically overloaded;
- `elbow_assist` residual use remains small;
- `shoulder_assist` stabilizes the fixed shoulder without affecting elbow
  conclusions.

## 13. Research Claim

The correct claim for this model is:

```text
The recommended Stage-2 model is an elbow-focused OpenSim arm26 variant with a
validated brachioradialis path and a physically represented 2 kg hand load. The
four shoulder muscles (DELT_ant/post, PECT, LAT) are removed because their
simplified shoulder paths are not physiologically valid for optimization and do
not cross the elbow; the fixed shoulder is held by the shoulder_assist actuator.
This provides a defensible biomechanical model for studying loaded repetitive
elbow flexion and generating simulation-based muscle/torque labels.
```
